To make this crawler into production, a lot of things needed to be changed. Firstly at the moment backend and frontend works as a one project this must change for several reasons. Another thing is project have some hardcoded configs a config file can be used to chnage and save those. At the moment project uses SQLite as DB but as the data volume increases using this can cause a lot of problems another DB like postgresql can be used to improve performance. 

Project also lacks a lot of error handling/monitoring options. If anything fails for some reason it stays as "some reason" this can be improved by logging (at least) into console. Also some features have obvious vulnerabilities for example there is no hard limit to what numbers are written inside start crawling page. Some guards and safety fauters would work well. Probably anyone would overwhelm the website by requests or can easily access database. There are a lot a more work needs to be done before deployment but as a 5 to 6 hour vibecoded project this is enough.
